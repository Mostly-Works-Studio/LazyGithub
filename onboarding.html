Welcome to LazyDeploy 🚀

Two quick steps and you'll be triggering builds and deployments directly from GitHub PR comments — no tab switching, no copy-pasting.

1
Create a GitHub Personal Access Token

LazyDeploy calls the GitHub API to read PR details, post build comments, and dispatch workflow runs. It needs a token to do this on your behalf.

Required scopes

  • repo — read PR details, post comments, create deployments, and trigger repository dispatch events
  • workflow — dispatch GitHub Actions workflows

Note: If your organisation enforces SAML SSO, you must also authorise this token for SSO after creating it, otherwise API calls to that org will be rejected.

2
Paste your GitHub token

Paste the token you just created — LazyDeploy validates it instantly.

3
Set up your configuration

Configure which repos get buttons, what actions they trigger, and how they're labelled. You can always tweak this later from the extension settings.

✓

You're all set!

LazyDeploy is active. Open any configured repo's PR to see your buttons in action.
