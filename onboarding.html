Welcome to LazyDeploy 🚀

Two quick steps and you'll be triggering builds and deployments directly from GitHub PR comments — no tab switching, no copy-pasting.

1
Create a GitHub Personal Access Token

LazyDeploy calls the GitHub API to read PR details, post build comments, and dispatch workflow runs. It needs a token to do this on your behalf.

  • repo — read PR details and post build trigger comments
  • workflow — dispatch GitHub Actions workflows
2
Paste your GitHub token

Paste the token you just created — LazyDeploy validates it instantly.

Want to configure workflows or button labels too?

3
Open any GitHub Pull Request

LazyDeploy scans PR comment threads for version strings and injects buttons automatically. Here's what you'll see:

🔨 Build Deploy to Release Deploy to Prod

The Build button appears in the PR header. Deploy buttons appear next to version strings in CI comments — hover a version link to reveal them. Workflow files, inputs, and button labels are all configurable.

✓

You're all set!

LazyDeploy is active on all GitHub PR pages. Head over to a PR to try it out.
