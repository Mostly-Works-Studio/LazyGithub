Get started with LazyGitHub

Three quick steps to add shortcuts & automations to your GitHub PRs — post comments, dispatch workflows, create deployments, all without leaving the page.

1
Create a GitHub Personal Access Token

LazyGitHub uses the GitHub API to read PR details, post comments, dispatch workflows, and create deployments — all on your behalf. It needs a Personal Access Token with repo and workflow scopes. Your token is stored locally in your browser and never sent to any server.

Required scopes

  • repo — read PR details, post comments, create deployments, and trigger repository dispatch events
  • workflow — dispatch GitHub Actions workflows

Note: If your organisation enforces SAML SSO, you must also authorise this token for SSO after creating it, otherwise API calls to that org will be rejected.

2
Paste your GitHub token

Paste the token you just created — LazyGitHub validates it instantly.

3
Set up your configuration

Configure which repos get buttons, what actions they trigger, and how they're labelled. You can always tweak this later from the extension settings.

✓

You're all set!

LazyGitHub is active. Open a PR in any configured repo to see your buttons. Head to settings anytime to add more actions or adjust your config.
