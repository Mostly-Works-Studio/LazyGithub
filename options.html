LazyGitHub

Build one-click PR actions across your repos. Set global defaults, then override by group or repo. Changes take effect after reloading your GitHub tab.

▾ GitHub Personal Access Token ✓ Connected

Generate a token at github.com/settings/tokens with the scopes listed below.

Required token scopes

  • repo — read PR details, post comments, create deployments, and trigger repository dispatch events
  • workflow — dispatch GitHub Actions workflows

Note: If your organisation enforces SAML SSO, you must also authorise this token for SSO after creating it, otherwise API calls to that org will be rejected.

Actions on PR

Buttons in the PR header that post comments, dispatch workflows, create deployments, or fire repository events. Use {branchName}, {prTitle}, {prNumber}, {repo}, or any extracted variable as placeholders. Add {input:"Label"} to prompt for user input at click time. Drag ⠿ to reorder.

Actions on PR Comment

Buttons that appear next to matching text in PR comments (hover to reveal, auto-hide after 5 s). Extract values like version numbers or hashes from the comment and use them as placeholders in action fields. Add {input:"Label"} to prompt for user input at click time. Drag ⠿ to reorder.

▾ Additional Settings
Stacks

Group related actions under a shared dropdown to reduce clutter — e.g. group "Build", "Test", "Deploy" under a "Release" button. Assign stacks to individual actions from inside each action card. One action can belong to multiple stacks.
