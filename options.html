LazyDeploy

Configure your GitHub token and workflow settings to start deploying directly from PR pages.

▾ GitHub Personal Access Token ✓ Connected

Generate a token at github.com/settings/tokens with the scopes listed below.

Required token scopes

  • repo — read PR details, post comments, create deployments, and trigger repository dispatch events
  • workflow — dispatch GitHub Actions workflows

Note: If your organisation enforces SAML SSO, you must also authorise this token for SSO after creating it, otherwise API calls to that org will be rejected.

Configure actions, tokens, and per-repo overrides. Click any section header to expand it. Use { } JSON to export your config or import one from a teammate. Reload the GitHub tab after saving for changes to take effect.

Actions on PR

Buttons injected into the PR header. Drag the ⠿ handle to reorder. Use {branchName}, {prTitle}, {prNumber}, {repo}, or any custom token in action fields. Use {ask:"Label"} to prompt the user for input at click time.

Actions on PR Comment

Buttons shown next to matching links in PR comments. Define tokens to extract from the comment, then use {tokenName} in action fields. Use {ask:"Label"} to prompt for input at click time. Drag the ⠿ handle to reorder.
