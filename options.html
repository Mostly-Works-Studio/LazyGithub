Setup

GitHub token ✓ Connected

LazyGitHub uses a personal access token to talk to the GitHub API on your behalf — reading PR details, posting comments, and dispatching workflows. It's stored only in your browser's synced extension storage.

Required token scopes

  • repo — read PR details, post comments, create deployments, and trigger repository dispatch events
  • workflow — dispatch GitHub Actions workflows

Generate one at github.com/settings/tokens. If your organisation enforces SAML SSO, also authorise the token for SSO, otherwise calls to that org are rejected.

Global · PR header

PR Actions

Buttons injected into the PR header. Each can post a comment, dispatch a workflow, create a deployment, or fire a repository event.

Use {branchName}, {prTitle}, {prNumber}, {repo}, or any extracted variable as placeholders. Add {input:"Label"} to prompt for a value at click time. Drag ⠿ to reorder.

Global · PR comments

Comment Actions

Buttons that appear next to matching text inside PR comments (hover to reveal, auto-hide after 5 s). Extract values like version numbers or commit hashes from a comment and reuse them as placeholders.

Add {input:"Label"} to prompt for a value at click time. Drag ⠿ to reorder.

Global · fine-tuning

Advanced

Optional settings that shape how buttons are grouped, which repos LazyGitHub runs on, and what the on-page info box shows.

Stacks

Group related actions under a shared dropdown to reduce clutter — e.g. put "Build", "Test", "Deploy" under one "Release" button. Assign stacks from inside each action card. One action can belong to multiple stacks.

Overrides · repo groups

Groups

Override the global config for a set of repos matched by name or regex. First matching group wins.

global ─▶ group ─▶ repo
each layer overrides the one before · most specific wins

Patterns in Match repos are tested top-to-bottom as an exact name, then as a regex. Toggle a section to override global config — unselected sections are inherited. Choose Replace to swap inherited actions out, or Extend to append after them. Enabling an override auto-fills from the current global config. Extraction patterns are global-only.

Overrides · single repo

Repos

Override the config for one specific repository. The most targeted layer — beats both global and any matching group.

global ─▶ group ─▶ repo
each layer overrides the one before · most specific wins

Exact repo name only — format owner/repo, no regex. Enabling an override auto-fills from the resolved parent config (global + matching group). A repo listed here is never excluded by repoFilter.
